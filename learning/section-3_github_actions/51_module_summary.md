### Notes Section (Main Notes)

**1. Module Summary**
- **Core Components**
  - **Workflows**: Define Events + Jobs
  - **Jobs**: Define Runners + Steps
  - **Steps**: Do the actual work
- **Events / Triggers**
  - Broad variety of events (repository-related and other)
  - Workflows have at least one (but possible more) event(s)
- **Defining Workflows**
  - `.github/workflows/<file>.yml` (on GitHub or locally)
- **Runners**
  - Servers (machines) that execute the jobs
  - Pre-defined Runners (with different OS options) exist.
  - You can also create custom Runners.
- **Workflow Execution**
  - Workflows are executed when triggered by defined events.
  - GitHub provides detailed insights into job execution (+ logs)
- **Actions**
  - You can run shell commands
  - But you can also use pre-defined Actions (official, community or custom)

### Summary Section (Summary of Notes)

GitHub is a cloud-based platform that provides Git repository hosting and various services to facilitate version control and collaboration among developers. Key features include cloud storage for Git repositories, enabling backup and teamwork across different machines, as well as tools for code management and collaboration, such as issues, projects, and pull requests.
