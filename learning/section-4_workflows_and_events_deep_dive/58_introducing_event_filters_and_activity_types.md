### Notes Section (Main Notes)

**1. Event Activity Types and Filters**

- **Event Activity Types**:
  - Refer to the specific actions that can trigger workflows in GitHub Actions. These include activities like pushing code, opening pull requests, and more.
- **Event Filters**:
  - Allow you to specify conditions under which a workflow should run based on the event data. This can include filtering by branch, tag, or other event properties.
  - By using event filters, you can ensure that workflows are only triggered for relevant events, improving efficiency and reducing unnecessary runs.
